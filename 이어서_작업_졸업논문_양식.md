## 2. 제작 파이프라인 (중요 — 재생성 시)
HTML→hwpx 직접변환은 줄간격 폭증으로 27p 실패함. 그래서 **정밀 DOCX 경로**를 씀:
```
paper_b5.html  --(code/build_docx.py)-->  임창우_졸업논문(docx).docx  --(한글 COM)-->  hwpx/PDF
```
- `code/build_docx.py`: paper_b5.html 파싱 → python-docx로 B5·바탕10.5pt(현재 11pt는 hwpx에서 수정)·줄간격160%·front matter·십진번호·표6·그림·차례(페이지번호 PAGES dict)·푸터페이지번호(국문요약=1, 섹션분리) 생성
- 페이지 매핑: PDF 렌더 후 각 절 페이지 찾아 build_docx.py의 `PAGES` dict 갱신 (2-pass)

### ⚠️ 절대 주의
**사용자가 한글에서 hwpx를 직접 손봤음** (제목 16/14pt, 띄어쓰기, 영문제목, 인준서 ㊞, 11pt 등). → **build_docx.py로 재생성하면 사용자 수정이 날아감.** 변경은 반드시 **그 hwpx에 직접**(한글 COM 또는 XML 수술).

## 3. 한글 COM 편집 패턴 (PowerShell, 검증됨)
```powershell
Get-Process | ? { $_.Name -match 'Hwp|Hncom' } | Stop-Process -Force   # 잔존 프로세스 먼저 종료 (안 하면 SaveAs=False)
$hwp = New-Object -ComObject "HWPFrame.HwpObject"
try { $hwp.RegisterModule("FilePathCheckDLL","FilePathCheckerModule") } catch {}
$hwp.Open($f, "HWPX", "")
# 찾기·바꾸기:
$fr=$hwp.HParameterSet.HFindReplace; $hwp.HAction.GetDefault("AllReplace",$fr.HSet)
$fr.FindString="X"; $fr.ReplaceString="Y"; $fr.IgnoreMessage=1; $fr.Direction=0; $fr.ReplaceMode=1
$hwp.HAction.Execute("AllReplace",$fr.HSet)   # 반환값 False여도 실제 적용될 수 있음 → 결과 확인 필수
# 쪽 나눔 삽입: MoveDocBegin → RepeatFind로 위치 → MoveParaEnd → Run("BreakPage")
$hwp.SaveAs($f,"HWPX",""); $hwp.SaveAs($pdf,"PDF","")   # PDF로 검증
$hwp.Quit()
```
- 글자크기 변경은 XML 수술도 가능: hwpx 압축풀어 `Contents/header.xml`의 charPr `height` 변경(1050=10.5pt, 1100=11pt), mimetype 무압축 첫 항목으로 재패키징.

## 4. 검증 방법
- 렌더: 한글 COM `SaveAs(pdf,"PDF")` 또는 `chrome --headless --print-to-pdf`(HTML B5)
- 페이지 확인: PyMuPDF(fitz)로 페이지 몽타주/텍스트 추출. (한글 변환 PDF는 한글 텍스트 추출 잘 됨)
- 한글 텍스트엔 ㊞(도장)·−(U+2212)가 `?`로 추출됨 → 실제 PDF/hwpx는 정상 (시각 확인으로 판별)
